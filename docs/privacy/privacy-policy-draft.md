# URAI Privacy Policy Draft

This draft is for product and engineering review before professional legal review.

## What URAI is

URAI is a personal reflection and symbolic life-mapping app. It helps users organize memories, moods, rituals, patterns, and optional AI Companion conversations through privacy controls.

## What URAI is not

URAI is not a medical device, therapist, emergency service, diagnostic tool, legal advisor, financial advisor, lie detector, or surveillance tool.

## Data users choose to provide

Users may choose to provide account details, settings, onboarding choices, reflections, ritual information, exports, and other app state. Some features may use optional summaries from opened layers.

## Passive data layers

URAI may include passive or contextual layers only when the user opens those layers. Sensitive passive layers should stay closed by default.

## Passport permissions

Passport is the control center for what URAI may use, sync, export, or include in AI context. Closed layers should remain closed.

## AI Companion context

The AI Companion should receive only approved summaries from opened layers. It should not receive raw sensitive data by default, and it should not use closed layers.

## Audio and transcripts

Sound playback is different from audio capture. Sound changes what URAI plays. Audio capture or transcription controls what URAI may listen to or transcribe and requires separate explicit permission.

## Location

Location is sensitive. URAI should prefer summaries and should not use exact location in AI context unless the user explicitly allows it.

## Gmail and calendar integrations

Gmail and calendar context require explicit connection and permission. Summaries are preferred. URAI should not access or use content from these integrations unless the user allows it.

## Health-adjacent data

Health-adjacent data is sensitive. URAI does not diagnose, treat, or provide medical advice.

## Shadow and sensitive reflections

Shadow stays closed by default. Shadow reflections are reflections, not verdicts. Shadow content should not appear in notifications or exports unless explicitly allowed and reviewed.

## Legacy archive

Legacy stays closed by default. Nothing is carried forward unless the user chooses. Legacy should use user-approved summaries.

## Exports

Exports require review before anything leaves URAI. Hidden, sealed, or blocked items should not export. Export filenames should avoid private identifiers.

## Notifications

Notifications are off by default or limited by user settings. Notification copy should stay generic and should not reveal sensitive content.

## Cloud sync

Cloud sync requires sign-in and user choice. Sync copies only approved URAI state to the user account. Cloud sync does not override Passport.

## Local-only mode

URAI may be used in local-only mode where supported. Local-only data stays on the device unless the user chooses to sync or export.

## Account deletion

Deletion controls apply to data URAI controls. This does not delete data from third-party services. Some deletion flows may mark data pending deletion until backend processing is complete.

## Data retention

Retention depends on the layer and user settings. Users should be able to close layers, reset local data, and request deletion of data URAI controls.

## Children and minors

URAI is not intended for children without appropriate legal review, age gating, and guardian consent flows where required.
