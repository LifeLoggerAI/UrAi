# Public Receipts Page Spec

## Purpose

The receipts page should make the URAI public story credible without overclaiming.

It should answer four questions:

1. What is URAI?
2. What exists now?
3. What is verified?
4. What is still gated?

## Hero copy

URAI turns scattered personal context into a spatial AI memory world.

## Founder line

Adam Clamp is the founder of URAI Labs, building spatial AI memory systems and user-owned personal intelligence infrastructure.

## Page sections

### 1. Product proof

Explain the public demo, memory-to-world direction, Life Map surface, companion route, privacy and terms pages, and early-access flow.

### 2. System proof

Link to architecture docs, repo maps, system registry, route matrix, release matrix, and proof spine.

### 3. Release proof

Explain that production certification requires exact passing SHA, deployment receipt, rollback SHA, route proof, status receipt, screenshots, privacy gates, and provider/device receipts when applicable.

### 4. Privacy proof

Link to privacy pages and evidence boundaries. Explain that personal memory systems must be consent-first and must not expose private user content in public evidence.

### 5. Search proof

Link founder, Adam Clamp, Adam Cohagen Clamp, URAI Labs, URAI Spatial, technology, ecosystem, and press pages.

## Public-safe status wording

URAI has public repositories, source docs, launch-gate docs, and evidence logs. Production certification remains gated until exact deployment and verification receipts exist.

## Do not say

Do not say complete, certified, clinically validated, legally approved, patent issued, provider-backed, device-certified, or advisor-endorsed unless the matching public receipt exists.
