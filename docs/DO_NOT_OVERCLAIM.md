# Do Not Overclaim

Updated: 2026-06-25

The URAI ecosystem has multiple repos with strong scaffolding, historical `done` language, or partial live surfaces. That is not enough to claim production readiness.

## Hard Rules

- Do not treat `UrAi-Dev` as production truth.
- Do not treat `UrAiProd` as a live product source.
- Do not call a repo production-ready because a README says `done`, `locked`, `ship`, or `production` without current evidence.
- Do not activate passive sensing, therapy-adjacent behavior, outbound comms, or data monetization without `urai-privacy` proof.
- Do not wire roadmap repos into `UrAi` just because environment variables already exist for them.
- Do not claim a domain is live if current HTTP headers show a different platform serving it.

## Current Known Overclaim Risks

- custom domains that still route to Squarespace
- repos with evidence files still marked `Blocked`
- services that are operationally ready in code but not proven live
