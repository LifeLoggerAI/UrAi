## Collection map

The Genesis collection contract includes `users`, `passports`, `permissionGrants`, `passiveSignals`, `audioLogs`, `transcriptSegments`, `voiceIdentities`, `socialProfiles`, `relationshipSignals`, `moodStates`, `moodForecasts`, `mentalLoadSnapshots`, `shadowCognitionMetrics`, `obscuraPatterns`, `recoveryEvents`, `rituals`, `narratorInsights`, `councilStates`, `companionStates`, `memoryStars`, `memoryConstellations`, `memoryBlooms`, `lifeChapters`, `mirrorReplays`, `emotionalWeatherFields`, `exportArtifacts`, and `adminAuditEvents`.

## Integration posture

Genesis additions are designed to be additive and compatible with existing V1 routes. Any system not fully connected to a production model or asset pipeline must remain safely scaffolded and should not claim live sensing in the public UI.
