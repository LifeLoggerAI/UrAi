# Privacy Review Brief

This is not legal advice.

## Privacy-First Principles

* Sample public demo before private data.
* Permission-first through Passport.
* Sensitive features off by default.
* User-controlled opening of protected layers.
* Clear separation between demo data and future private data.

## Demo Data

The public demo uses sample data only and should not expose private user data or connect to private integrations.

## Waitlist and Feedback

Waitlist collection should be limited to contact information the user chooses to submit. Feedback should not request sensitive personal details.

## Review Requests

Counsel should review privacy notice needs, consent language, retention language, deletion language, platform policy expectations, and future sensitive feature requirements.
