# Consent and Passport Map

This is not legal advice.

Passport is the visible consent and permission interface for URAI Genesis.

## Consent Model

* **Permission-First:** Features are off by default until the user explicitly enables them.
* **Transparency:** Passport clearly shows what is open, closed, or deferred.
* **Granularity:** Users should have granular control over different data types and integrations.

## Passport States

| State | Description | User Action |
| --- | --- | --- |
| **Locked** | Feature is not available and requires future review. | None. User is informed it is deferred. |
| **Closed** | Feature is available but not enabled. | User must explicitly opt-in to open. |
| **Open** | Feature is active. | User can close it at any time. |

## Review Focus

Counsel should review the clarity of consent language, the user flow for enabling and disabling features, and whether the Passport model meets platform and legal requirements.
