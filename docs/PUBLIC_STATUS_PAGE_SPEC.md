# URAI Public Status Page Spec

## Goal

Create one place where anyone can understand URAI system state without reading private operational details.

## Status categories

Each item should show one state:

- Unknown
- Scaffolded
- Built
- Verified
- Deployed
- Live-proven
- Provider-backed
- Device-certified
- Blocked

## Public sections

### Product

Current demo routes, experience surfaces, and public availability.

### Runtime

System health summaries without exposing private data.

### Release

Tested SHA, deployed SHA, rollback state, and evidence links when available.

### Privacy

Trust controls, consent boundaries, and privacy documentation.

### Roadmap

Future layers clearly separated from current verified capabilities.

## Rule

No green status without receipts.

A public status page should increase trust by showing both progress and remaining gates.
